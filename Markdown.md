HTML семантика та структура
Що це: семантичні теги описують роль блоків на сторінці — не тільки як вони виглядають, а що означають. Це важливо для SEO, доступності (screen readers) і логіки проєкту.

Основні теги і коли їх використовувати

<header> — шапка сторінки або секції. Містить логотип, навігацію, пошук.
Приклад: шапка сайту з логотипом і меню.

<main> — основний контент сторінки. Має бути один на документ.
Приклад: блок зі статтею або списком товарів.

<footer> — підвал: контакти, копірайт, посилання на політику конфіденційності.

<section> — тематичний розділ (наприклад, «Послуги»). Використовуй, коли блок має заголовок і логічно відокремлений.

<article> — самостійна публікація (новина, пост). Може існувати поза контекстом.

<aside> — побічний контент: віджети, рекомендації, реклама.

Практика

html

<header>
  <h1>Магазин</h1>
  <nav>...</nav>
</header>
<main>
  <section>
    <article>
      <h2>Огляд товару</h2>
      <p>Опис...</p>
    </article>
  </section>
  <aside>Рекомендації</aside>
</main>
<footer>© 2026</footer>
Порада: семантика — перше, що робиш при верстці. Вона полегшує тестування, SEO і доступність.

Текст, списки, посилання, зображення
Заголовки і текст

<h1>–<h6> — ієрархія заголовків. Один <h1> на сторінку (загальний заголовок).

<p> — абзац.

<span> — інлайн‑контейнер для стилів або JS (не змінює структуру).

<strong>, <em> — семантичне виділення (важливість, курсив).

Списки

<ul> — маркований список; <ol> — нумерований; <li> — елемент.
Використовуй списки для меню, чек‑листів, кроків.

Посилання

<a href="..."> — посилання. Для зовнішніх посилань з target="\_blank" додавай rel="noopener noreferrer".

Зображення

<img src="..." alt="..."> — обов’язково alt (доступність, SEO).

Для адаптивності використовуй srcset/sizes або <picture> (див. розділ про медіа).

Вкладеність

Дотримуйся правил: не вставляй блокові елементи всередину <p>. <ul> всередині <li> — нормально.

Приклад:

html

<p>Наші послуги:
  <ul>
    <li>Дизайн</li>
    <li>Розробка</li>
  </ul>
</p>
<!-- Неправильно: <div> всередині <p> -->
Медіа та адаптивні зображення
Формати

JPEG — фото; PNG — прозорість; WebP/AVIF — сучасні, менші за розміром; SVG — векторні іконки/логотипи.

srcset і sizes

Дозволяють браузеру вибрати оптимальний файл залежно від розміру екрана і щільності пікселів.

html
<img src="photo-800.jpg"
     srcset="photo-480.jpg 480w, photo-800.jpg 800w, photo-1200.jpg 1200w"
     sizes="(max-width:600px) 480px, 800px"
     alt="Опис">
w — ширина зображення; x — множник щільності (наприклад image@2x.png 2x).

<picture>

Використовуй, коли потрібно змінювати формат (WebP) або подавати зовсім інше зображення для певних умов.

html
<picture>

  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Опис">
</picture>
Lazy loading

Додавай loading="lazy" для зображень нижче згину сторінки.

Відео та аудіо

<video controls poster="poster.jpg" playsinline muted> — для мобільних playsinline важливий.

Формати: MP4 (H.264), WebM.

Порада: роби ретинізацію (2x) для ключових елементів, але не для всього; використовуй CDN і компресію.

CSS основи: підключення, селектори, специфічність
Підключення
Inline: style="..." — швидко, але не для продакшену.

Internal: <style> в <head> — для сторінки.

External: <link rel="stylesheet" href="style.css"> — рекомендовано.

Селектори і приклади
Тег: p { color: #333; }

Клас: .btn { padding: 8px; }

ID: #header { height: 60px; } (уникай для стилізації, краще класи)

Атрибут: input[type="email"] { }

Нащадок: .nav a { }

Дитина: .list > li { }

Псевдокласи: :hover, :focus, :nth-child(2).

Каскадність і специфічність
Правило: inline > ID > class/attribute/pseudo-class > tag.

!important — крайній засіб, уникай.

CSS змінні
css
:root {
--accent: #ff6600;
}
.btn { background: var(--accent); }
Зручно для тем, швидких змін кольорів.

Шрифти та типографіка
Підключення

Google Fonts: <link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">

Локальні: @font-face { font-family: 'MyFont'; src: url('myfont.woff2') format('woff2'); }

Властивості

font-family, font-size, font-weight, line-height, letter-spacing, text-transform.

Практика

Використовуй font-display: swap; у @font-face для кращого UX.

Для масштабування шрифтів застосовуй rem і clamp():

css
html { font-size: 16px; }
h1 { font-size: clamp(1.5rem, 2.5vw, 2.5rem); }
Layout: Flexbox і Grid
Flexbox — коли використовувати
Одновимірні макети (ряд або стовпець): навігація, ряд карток, центровані блоки.

Контейнер

css
.container {
display: flex;
flex-direction: row; /_ row | column _/
justify-content: space-between; /_ вирівнювання по головній осі _/
align-items: center; /_ по перехресній осі _/
gap: 16px;
}
Елемент

css
.item {
flex: 1 1 200px; /_ flex-grow flex-shrink flex-basis _/
order: 2; /_ змінює порядок візуально _/
align-self: flex-start; /_ індивідуальне вирівнювання _/
}
Grid — коли використовувати
Двовимірні макети: складні сітки, лейаути з рядами і колонками.

css
.grid {
display: grid;
grid-template-columns: repeat(3, 1fr);
gap: 20px;
}
Порада: комбінуй Grid для загальної структури і Flexbox для внутрішніх компонентів.

Позиціонування, overflow, z-index
Типи позиціонування

static — за замовчуванням.

relative — зміщення від нормального місця (залишає простір).

absolute — позиціонується від найближчого предка з позицією (не static).

fixed — відносно вікна браузера (корисно для фіксованих шапок).

sticky — прилипає при прокрутці між relative і fixed.

z-index

Працює для позиціонованих елементів; більший z-index — вище шар.

overflow

visible, hidden, scroll, auto. Використовуй overflow: auto для скролу в контейнерах.

Приклад модального вікна

css
.modal {
position: fixed;
top: 50%;
left: 50%;
transform: translate(-50%, -50%);
z-index: 1000;
}
.backdrop { position: fixed; inset: 0; background: rgba(0,0,0,0.5); }

Фонові властивості

Градієнти

Псевдоелементи

css

SVG

Inline SVG дозволяє стилізувати елементи через CSS/JS.

Спрайт SVG: зберігає іконки в одному файлі і використовує <use> для вставки.

html
<svg viewBox="0 0 24 24"><use href="#icon-search"></use></svg>
Порада: для іконок краще SVG; для фото — WebP/AVIF.

Трансформації, переходи, анімації
Transform

transform: translate(), rotate(), scale(), skew() — змінює вигляд без перерахунку layout (GPU‑friendly).

Transition

transition: transform 0.3s ease; — плавні зміни при зміні властивостей.

Animation

@keyframes для складних циклічних анімацій.

css
@keyframes fadeIn {
from { opacity: 0; transform: translateY(10px); }
to { opacity: 1; transform: translateY(0); }
}
.card { animation: fadeIn 0.5s ease forwards; }
Порада: анімуй transform і opacity для кращої продуктивності; уникай анімацій, що змінюють layout (width/height) часто.

В justify-content у Flexbox є кілька варіантів розподілу простору між елементами. Розглянемо три, які ти назвав:

Значення	Як працює	Візуальний ефект
space-between	Елементи розташовуються так, що перший прилягає до лівого краю контейнера, останній — до правого, а весь вільний простір рівномірно розподіляється між елементами.	Краї контейнера «притиснуті», проміжки тільки між елементами.
space-around	Кожен елемент отримує однаковий простір зліва і справа. В результаті відступи між елементами удвічі більші, ніж відступи від країв контейнера.	Є «половинні» відступи біля країв і повні між елементами.
space-evenly	Вільний простір розподіляється повністю рівномірно: однакові відступи і між елементами, і від країв контейнера.	Всі проміжки однакові, включно з краями.
Приклад коду:

css
.container {
  display: flex;
  height: 100px;
  border: 1px solid #000;
}

.space-between { justify-content: space-between; }
.space-around  { justify-content: space-around; }
.space-evenly  { justify-content: space-evenly; }
Практичне використання:

space-between — добре для меню, де перший і останній елемент мають бути «притиснуті» до країв.

space-around — підходить для блоків, де потрібні симетричні відступи, але не однакові з краями.

space-evenly — сучасний варіант, коли хочеш ідеально рівномірний розподіл, включно з краями контейнера.

Таким чином, різниця головно у тому, як поводяться краї контейнера:

between — без відступів на краях,

around — половинні відступи на краях,

evenly — повні відступи на краях.



В align-items ми керуємо тим, як елементи у flex‑контейнері вирівнюються по перехресній осі (зазвичай вертикальній, якщо flex-direction: row). Це відрізняється від justify-content, який працює по головній осі (горизонтальній у рядку).

Ось основні значення:

Значення	Як працює	Візуальний ефект
flex-start	Всі елементи вирівнюються до верхнього краю контейнера (початок перехресної осі).	Елементи «притиснуті» вгору.
flex-end	Всі елементи вирівнюються до нижнього краю контейнера (кінець перехресної осі).	Елементи «притиснуті» вниз.
center	Елементи вирівнюються по центру перехресної осі.	Всі елементи по вертикалі в центрі контейнера.
stretch (значення за замовчуванням)	Елементи розтягуються, щоб заповнити весь доступний простір по перехресній осі.	Висота елементів = висота контейнера.
baseline	Елементи вирівнюються по базовій лінії тексту.	Текст у різних елементах стає на одну лінію.
Приклад коду:

css
.container {
  display: flex;
  height: 200px;
  border: 1px solid #000;
}

.start   { align-items: flex-start; }
.end     { align-items: flex-end; }
.center  { align-items: center; }
.stretch { align-items: stretch; }
.base    { align-items: baseline; }
Практичне використання:

flex-start — коли треба, щоб усі блоки були «притиснуті» до верху (наприклад, меню).

flex-end — для футерів або коли елементи мають бути вирівняні вниз.

center — часто використовується для вертикального центровання.

stretch — зручно для карток, які мають однакову висоту.

baseline — корисно для вирівнювання текстових блоків різного розміру.

Таким чином, різниця між ними у тому, де саме по вертикалі (або по перехресній осі) елементи вирівнюються: верх, низ, центр, розтягування чи базова лінія тексту .




У властивості flex-direction ми задаємо, в якому напрямку розташовуються елементи у flex‑контейнері. Це визначає головну вісь (main axis), а відповідно й те, як працюють justify-content та align-items.

Ось усі варіанти:

Значення	Як працює	Візуальний ефект
row	Елементи йдуть зліва направо (як текст у більшості мов).	Стандартний горизонтальний ряд.
row-reverse	Елементи йдуть справа наліво. Перший елемент стає крайнім правим.	Горизонтальний ряд, але порядок зворотний.
column	Елементи йдуть зверху вниз.	Вертикальний список.
column-reverse	Елементи йдуть знизу вверх. Перший елемент стає внизу.	Вертикальний список у зворотному порядку.
Приклад коду:

css
.container {
  display: flex;
  flex-direction: row; /* або row-reverse, column, column-reverse */
}
Практичне використання:

row — стандарт для меню, навігації, карток.

row-reverse — коли треба швидко поміняти порядок без зміни HTML (наприклад, кнопки «Назад» і «Далі»).

column — для форм, списків, мобільних макетів.

column-reverse — коли треба відображати нові елементи внизу, але логічно вони перші (наприклад, чат).

Важливо:

При row і row-reverse головна вісь — горизонтальна, перехресна — вертикальна.

При column і column-reverse головна вісь — вертикальна, перехресна — горизонтальна. Це впливає на те, як працюють justify-content (по головній осі) і align-items (по перехресній).



У Flexbox властивість order визначає порядок відображення елементів у контейнері, незалежно від того, як вони йдуть у HTML. За замовчуванням усі елементи мають order: 0.

Як працює
Чим менше значення order, тим раніше елемент з’явиться у візуальному порядку.

Елементи з однаковим order відображаються у тому порядку, як вони йдуть у HTML.

Можна використовувати як позитивні, так і негативні значення.

Приклад
html
<div class="container">
  <div class="item a">A</div>
  <div class="item b">B</div>
  <div class="item c">C</div>
</div>
css
.container {
  display: flex;
}

.a { order: 2; }   /* піде третім */
.b { order: -1; }  /* піде першим */
.c { order: 1; }   /* піде другим */
Результат: порядок буде B → C → A, хоча в HTML вони йдуть A → B → C.

Практичне використання
Адаптивність: можна міняти порядок елементів на різних екранах без зміни HTML.

css
@media (max-width: 600px) {
  .sidebar { order: 2; }
  .content { order: 1; }
}
Інтерфейси: кнопки «Назад» і «Далі» можна переставити залежно від мови (LTR/RTL).

Тимчасове тестування: швидко змінити порядок блоків без редагування розмітки.

Важливо
order змінює лише візуальний порядок, але DOM залишається тим самим. Це означає, що для screen readers, SEO та скриптів порядок у HTML все одно важливий.

Використовуй order для стилізації, але не для критичної логіки (наприклад, не міняй порядок формових полів через order



Властивість align-self у Flexbox дозволяє окремому елементу перевизначити вирівнювання, яке задане для всіх через align-items. Тобто, якщо в контейнері всі елементи вирівняні, наприклад, по центру, то конкретному елементу можна задати інше вирівнювання.

Основні значення align-self
Значення	Як працює	Візуальний ефект
auto	Використовує значення з align-items контейнера.	Поведінка як у всіх інших елементів.
flex-start	Елемент вирівнюється до початку перехресної осі (зверху, якщо flex-direction: row).	Притиснутий до верхнього краю.
flex-end	Елемент вирівнюється до кінця перехресної осі (знизу).	Притиснутий до нижнього краю.
center	Елемент вирівнюється по центру перехресної осі.	Вертикально по центру контейнера.
stretch	Елемент розтягується, щоб заповнити весь простір по перехресній осі.	Висота елемента = висота контейнера.
baseline	Елемент вирівнюється по базовій лінії тексту.	Текст у різних елементах стає на одну лінію.
Приклад коду
html
<div class="container">
  <div class="item a">A</div>
  <div class="item b">B</div>
  <div class="item c">C</div>
</div>
css
.container {
  display: flex;
  align-items: center; /* всі елементи по центру */
  height: 200px;
  border: 1px solid #000;
}

.a { align-self: flex-start; }  /* піде вгору */
.b { align-self: flex-end; }    /* піде вниз */
.c { align-self: stretch; }     /* розтягнеться */
Практичне використання
Виділення окремого елемента — наприклад, одна кнопка в меню має бути вирівняна вниз, а решта по центру.

Адаптивність — можна змінювати вирівнювання конкретного блоку на різних екранах.

Тестування — швидко перевірити, як виглядає інший варіант вирівнювання без зміни всього контейнера.

Таким чином, align-items задає вирівнювання для всіх елементів контейнера, а align-self дозволяє перевизначити його для конкретного елемента.


Властивість flex-wrap у Flexbox визначає, чи можуть елементи переноситися на новий ряд/стовпець, якщо вони не вміщаються у контейнер. За замовчуванням елементи стискаються в один ряд, навіть якщо стає тісно.

Основні значення flex-wrap
Значення	Як працює	Візуальний ефект
nowrap (за замовчуванням)	Усі елементи залишаються в одному рядку/стовпці. Вони стискаються, щоб поміститися.	Один ряд, елементи можуть ставати дуже вузькими.
wrap	Елементи переносяться на новий ряд/стовпець, якщо не вистачає місця.	Кілька рядків, елементи зберігають розміри.
wrap-reverse	Те саме, що wrap, але нові рядки додаються у зворотному напрямку (наприклад, зверху вниз → знизу вверх).	Перенесення, але порядок рядків інший.
Приклад коду
css
.container {
  display: flex;
  flex-wrap: wrap; /* або nowrap, wrap-reverse */
  gap: 10px;
}
.item {
  flex: 0 0 200px; /* фіксована ширина */
  height: 100px;
  background: lightblue;
}
HTML:

html
<div class="container">
  <div class="item">1</div>
  <div class="item">2</div>
  <div class="item">3</div>
  <div class="item">4</div>
  <div class="item">5</div>
</div>
Практичне використання
nowrap — добре для навігаційних меню, де всі елементи мають бути в одному рядку.

wrap — для галерей, карток товарів, тегів, коли елементи повинні переноситися і зберігати розмір.

wrap-reverse — рідко використовується, але може бути корисним для специфічних макетів (наприклад, коли нові елементи мають з’являтися зверху).

Важливо
flex-wrap часто комбінують із flex-direction. Разом вони утворюють flex-flow:

css
.container {
  display: flex;
  flex-flow: row wrap; /* напрямок + перенесення */
}



Властивість flex-flow у Flexbox — це шорткат, який об’єднує дві властивості:

flex-direction (напрямок головної осі)

flex-wrap (чи можна переносити елементи на новий ряд/стовпець).

Синтаксис
css
flex-flow: <flex-direction> <flex-wrap>;
Приклади
css
.container {
  display: flex;
  flex-flow: row nowrap;      /* стандарт: елементи в ряд, без переносу */
}

.container {
  display: flex;
  flex-flow: row wrap;        /* елементи в ряд, але можуть переноситися */
}

.container {
  display: flex;
  flex-flow: column wrap;     /* елементи зверху вниз, можуть переноситися */
}

.container {
  display: flex;
  flex-flow: row-reverse wrap-reverse; /* справа наліво + перенесення у зворотному напрямку */
}
Практичне використання
row nowrap — дефолт, підходить для меню, навігації.

row wrap — для галерей, карток товарів, тегів, коли елементи мають переноситися.

column wrap — для вертикальних списків, які можуть розбиватися на кілька колонок.

row-reverse wrap — коли треба змінити порядок елементів без зміни HTML.

Важливо
Якщо ти часто задаєш і напрямок, і перенесення — зручніше писати flex-flow, ніж окремо flex-direction і flex-wrap.

За замовчуванням: flex-flow: row nowrap;.



Властивість align-content у Flexbox використовується для керування тим, як рядки (а не окремі елементи) розташовуються по перехресній осі, коли у контейнері є кілька рядків (тобто при flex-wrap: wrap). Якщо рядок лише один — ця властивість не має ефекту.

Основні значення align-content
Значення	Як працює	Візуальний ефект
flex-start	Усі рядки притискаються до початку перехресної осі (зверху, якщо flex-direction: row).	Рядки зібрані вгорі контейнера.
flex-end	Усі рядки притискаються до кінця перехресної осі (внизу).	Рядки зібрані внизу контейнера.
center	Рядки вирівнюються по центру перехресної осі.	Рядки зібрані посередині контейнера.
stretch (за замовчуванням)	Рядки розтягуються, щоб заповнити весь простір по перехресній осі.	Відстань між рядками автоматично розтягується.
space-between	Перший рядок притиснутий до початку, останній — до кінця, а простір між рядками розподілений рівномірно.	Рівні проміжки між рядками, але без відступів від країв.
space-around	Кожен рядок має однакові відступи зверху і знизу. Відступи між рядками удвічі більші, ніж відступи від країв.	«Половинні» відступи біля країв, повні між рядками.
space-evenly	Вільний простір розподіляється повністю рівномірно: однакові відступи між рядками і від країв.	Абсолютно рівні проміжки всюди.
Приклад коду
css
.container {
  display: flex;
  flex-wrap: wrap;       /* кілька рядків */
  align-content: space-between; /* або інше значення */
  height: 400px;
  border: 1px solid #000;
}
.item {
  flex: 0 0 100px;
  height: 100px;
  background: lightblue;
  margin: 5px;
}
Практичне використання
flex-start / flex-end — коли треба зібрати всі рядки вгорі або внизу контейнера.

center — для симетричного вирівнювання кількох рядків.

stretch — зручно для галерей, щоб рядки рівномірно заповнювали контейнер.

space-between / space-around / space-evenly — для рівномірного розподілу рядків у високому контейнері (наприклад, сітка карток).

👉 Головна різниця між align-items і align-content:

align-items вирівнює елементи всередині одного рядка.

align-content вирівнює рядки між собою, коли їх кілька.
